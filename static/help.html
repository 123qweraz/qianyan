千言输入法 完整帮助文档

一款专为程序员与效率极客设计的高性能拼音输入法

产品简介

属性说明
项目名称千言输入法 (Qianyan IME)
编程语言Rust (Edition 2021)
许可证MIT
版本0.1.0
支持平台Linux (evdev/IBus/Wayland)、Windows (TSF)

设计理念

千言输入法 的设计核心是「零干扰」与「极致效率」：

  • 位置固定：候选框默认固定在屏幕底部，不跟随光标，减少视觉抖动
  • 快捷切换：使用 Tab 键瞬间切换中英文
  • 辅助码定位：拼音重码时，用大写字母或笔画码精准定位，无需翻页

# 安装
sudo cp target/release/qianyan-ime /usr/local/bin/

权限配置（evdev 模式必需）

重要：evdev 模式需要读取输入设备和创建虚拟键盘的权限

# 1. 将当前用户加入 input 组
sudo usermod -aG input $USER

# 2. 创建 udev 规则
echo 'KERNEL=="uinput", GROUP="input", MODE="0660", OPTIONS+="static_node=uinput"' \
  | sudo tee /etc/udev/rules.d/99-qianyan-ime-uinput.rules

# 3. 重载 udev 规则
sudo udevadm control --reload-rules
sudo udevadm trigger

注意：加入 input 组后，必须注销并重新登录才能生效

后端选择

程序自动检测可用后端，优先级：evdev → IBus → Wayland

命令说明

qianyan-ime --backend=wayland强制 Wayland 原生协议

常见问题

Q1: 启动后键盘没反应？

原因：evdev 模式权限未正确配置。

解决：

  1. 确保用户已加入 input 组：运行 groups 确认
  2. 确保 udev 规则已创建：检查 /etc/udev/rules.d/99-qianyan-ime-uinput.rules
  3. 注销并重新登录（这步很重要！）

Q2: 候选窗不显示？

原因：Slint 渲染后端问题，或桌面环境兼容性问题。

兼容性情况：

桌面环境兼容性
KDE (Wayland)最佳
COSMIC最佳
Hyprland最佳
GNOME候选窗可能无法悬浮
XFCE候选窗可能无法悬浮

尝试切换渲染后端：

export SLINT_BACKEND=skia
qianyan-ime

Q3: Web 配置页面打不开？

检查步骤：

  1. 确认 qianyan-ime 进程在运行：ps aux | grep qianyan
  2. 检查端口 18765 是否被占用：netstat -tlnp | grep 18765
  3. 确认地址：http://localhost:18765（注意是 18765，不是 8765）

Q4: 如何卸载？

Linux：

# 删除程序文件
sudo rm /usr/local/bin/qianyan-ime
sudo rm /usr/share/applications/qianyan-ime.desktop
sudo rm /etc/udev/rules.d/99-qianyan-ime-uinput.rules

# 删除用户数据
rm -rf ~/.local/share/qianyan-ime

核心：英文辅助码

这是 千言输入法 的灵魂功能。在拼音重码极多（如同音字、简拼）时，您无需翻页，只需输入一个大写字母即可精准定位。

工作原理

输入法会自动扫描候选词对应的英文翻译/释义。当您输入一个大写字母时，它会匹配释义中第一个大写单词的首字母。

场景 A：解决同音字重码

输入 li，候选词可能有"里、离、礼、理..."。

  • 输入 liC ⮕ 礼 (Ceremony) 瞬间排到第一。
  • 输入 liL ⮕ 理 (Logic) 瞬间排到第一。

场景 B：简拼精准定位

输入 sh，候选词可能有"书、水、数、树..."。

  • 输入 shB ⮕ 书 (Book) 瞬间出现。
  • 输入 shW ⮕ 水 (Water) 瞬间出现。
  • 输入 shT ⮕ 树 (Tree) 瞬间出现。

技巧：该功能会自动跳过声调符号。例如翻译是 "ne Particle"，输入 P 即可精准匹配 "Particle"。

进阶：笔画辅助码 (SBSRF)

这是本引擎最核心的原创设计方案。它不仅可以作为拼音的辅助过滤码，更可以作为一套独立、高效的笔画输入法。其核心逻辑是：一键代表两个连续笔画。

设计思想：双笔一键

传统的笔画输入法（1-5数字键）效率低下，因为每输入一个笔画就需要点击一次按键。SBSRF 方案通过 5x5 的字母矩阵，让您每一次击键都能输入两个笔画，大幅提升了输入速度。

映射布局：5x5 视觉矩阵

每个按键代表两笔：行代表第 1 笔，列代表第 2 笔。

编码规则

输入类型编码规则
单字全码依次输入笔画对对应的字母
单字简码8笔以上汉字：前3键 + 最后1键
万能通配符Z 键匹配任意一个编码字母（2个笔画）
2字词第1字前2键 + 第2字前2键
3字词第1字首键 + 第2字首键 + 第3字前2键
4字及以上前3字首键 + 最后1字首键

统计特性

重码率极低：在全量 2 万字字典中，超过 95% 的编码是唯一的。即使在重码情况下，绝大多数编码也仅对应 2 个汉字。

笔画数分布：6 笔及以下的汉字仅占总量的约 6%。绝大多数汉字（7 笔及以上）通过 4 键编码即可实现精准区分。

专有名词原生隔离

在普通输入法中，输入 beijing 可能会混杂"背景"和"北京"。在 千言输入法 中：

输入匹配结果
beijing（全小写）普通词汇（如：背景）
Beijing 或 Be专有名词（如：北京）

这对于输入人名、地名、软件名非常有效，实现了索引层面的清爽隔离。

多方案混输

千言输入法 允许您自由组合活跃方案（如 中文+英文）：

  • 程序员友好：写注释时无需切换模式，直接输入 git, cargo, sudo
  • 英文自动补全：输入 com 即可选择 complete, command 等

在「系统设置」中启用多个输入方案即可体验。

长按与双击定制

双击快捷输入

快速点击同一个字母键两次，注入长韵母或常用短语：

双击注入
i, iing
u, ush

长按行为定制

按住字母键不放（约 400ms），触发预设操作。默认映射为该字母的大写形式。

配置提示：访问「键盘布局」设置，点击可视化键盘上的字母，即可分别为"双击"和"长按"分配自定义字符串。

智能导航与平卷互换

HJKL 导航模式

在输入拼音的过程中，按下 ` (反引号) 即可进入导航模式：

按键功能
H切换上一个候选词
L切换下一个候选词
J向下翻页
K向上翻页

按任意其他字母键将自动退出导航并继续拼写。

平卷舌一键互换

拼错平/卷舌音时无需删除重打。输入 zan 时发现应该是 zhan？只需按一下 / 键，系统会自动完成互换。

提示词上屏

按下 Shift + Space 可将候选词旁边的英文提示或注音直接上屏。

汉字笔画纠错手册

一、笔顺易错（最常写错顺序）

火 编码: iw 点→撇→撇→捺
方 编码: yv 点→横→横折钩→撇
为 编码: ic 点→撇→横折钩→点
片 编码: ra 撇→竖→横→横折
九 编码: q 撇→横折弯钩
万 编码: at 横→横折钩→撇
凸 编码: hmg 竖→横→竖→横折折折→横
凹 编码: mmg 竖→横折折→竖→横折→横

二、笔画数量/长短易错

具 编码: mggw 中间三横（不是两横）
冒 编码: mgmgg 上部是冃，两横不接框
肺 编码: qgfb 右边一竖到底（不多一横）
黄 编码: fhmfdy 下部是"由"（不是田）

三、出头/不出头易错

寻/雪 中横不出头
建/康 中横出头
临 编码: jtubg 左边是两竖（不是撇）

四、多撇/少撇易错

免/象/卑 第六笔撇通下来
畏/展/武 不要多一撇

五、点画易错

市 编码: ymh 顶部有点
沛/尧/步/纸 右侧无点
义 编码: iy 先写点
发/成 后写点
